# Squad Memory: Instagram Project Carousel

## Estilo de Escrita
- Tom: Premium, focado em resultados de negócio, autoridade técnica e clareza.
- Ganchos: Começar com métricas reais do projeto (ex: "50.000+ veículos", "98% satisfação").
- CTA: Preferência por "Comente [PALAVRA]" em vez de "Link na bio" — gera mais engajamento.
- Legenda: Máximo 1500 caracteres. Emojis no início de tópicos. Hashtags no final (máx 10).

## Design Visual
- Paleta validada: Dark Mode `#08090D` + Teal `#00D9A6` + Inter 900.
- Layout aprovado: Formato Split (imagem+texto) para slides de impacto, texto puro para conteúdo técnico.
- Screenshots reais obrigatórios: Capturar homepage, métricas e features do projeto em produção.
- Gradient overlay: `linear-gradient(to bottom, transparent 30%, #08090D 100%)` sobre imagens.
- Rodapé padrão: `{n}/{total}` à esquerda, `DU·SOFTWARE` à direita.

## Estrutura de Conteúdo
- Framework validado: Case Study Editorial (10 slides).
- Sequência aprovada: Capa → Problema → Solução → Tech Stack → Resultados → Como Funciona → Design → Features → Checklist → CTA.
- Cada projeto do portfólio gera exatamente 1 carrossel de 10 slides.

## Proibições Explícitas
- Proibido usar imagens de banco de imagens (stock photos).
- Proibido usar background branco ou claro.
- Proibido ganchos genéricos como "Conheça nosso novo app".
- Proibido legendas começando com "Nesse post vamos falar de...".

## Técnico (específico do squad)
- Browser screenshots: Sempre capturar 3+ screenshots reais do projeto via browser antes de criar slides.
- Renderização: Usar `document.write()` ou data URI para injetar HTML no browser e capturar PNG.
- Viewport de captura: 1080×1440 para slides, 1280×900 para screenshots de projeto.
- Imagens do projeto devem ser salvas no diretório do run antes de serem referenciadas nos HTMLs.
